Repository Live Architecture

Scalable Microservices on AWS (EC2/S3/IAM)

// Problem Statement

Local development environments lacked parity with production, causing "works on my machine" errors. The application struggled under simulated loads and required high manual intervention for provisioning and monitoring.

// Architecture Flow

Users ➔ NGINX Reverse Proxy ➔ Docker Container (Flask API) ➔ AWS S3 (Storage)

// DevOps Implementation

  • CI/CD Pipeline: Built a GitHub Actions workflow to run Python unit tests, linting, and automatically build/push Docker images to Docker Hub on every commit to main.
  • Docker Usage: Standardized the environment through multi-stage Dockerfiles, reducing payload image sizes by ~40% and isolating application dependencies.
  • Deployment Strategy: Deployed via SSH tunneling to a hardened AWS EC2 instance. Bound minimal-privilege AWS IAM roles for secure external S3 bucket reads without leaking API keys.

Repository Live Demo

Automated Delivery Pipeline for Local Business

// Problem Statement

A local bakery required a digital presence but lacked the technical know-how to deploy updates (like seasonal menus). Manual FTP uploads were causing downtime and data mismatches.

// Architecture Flow

Git Push ➔ GitHub Actions (Test & Validation) ➔ Vercel Edge CDN ➔ Global Users

// DevOps Implementation

  • CI/CD Pipeline: Developed robust GitHub Actions pipelines running static code analysis (Prettier/HTMLHint) to block broken code from entering production.
  • Infrastructure as Code (IaC): Programmed vercel.json to enforce strict caching policies and inject security headers (X-Frame-Options) unconditionally.
  • Deployment Strategy: Established Continuous Deployment (CD) integrated natively via Vercel Webhooks, achieving sub-10 second deployments and a 100% reduction in manual FTP effort.
